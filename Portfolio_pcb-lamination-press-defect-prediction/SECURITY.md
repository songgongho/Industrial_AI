# Security Policy

## Reporting Security Vulnerabilities

**Do not** open a public GitHub issue for security vulnerabilities!

Instead, please follow responsible disclosure:

### Reporting Steps

1. **Email**: Contact the maintainer at `contact@example.com`
2. **Subject**: `Security Vulnerability Report - [project name]`
3. **Content**: Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if applicable)

### What to expect

- **Acknowledgment**: We'll respond within 48 hours
- **Assessment**: We'll evaluate the severity
- **Timeline**: 
  - Critical: Patch within 7 days
  - High: Patch within 14 days
  - Medium/Low: Included in next release
- **Credit**: Full credit in security advisory (unless you prefer anonymity)

## Security Best Practices

When using this project:

✅ **DO**:
- Use the latest version
- Review the CHANGELOG for security fixes
- Keep your dependencies updated
- Use strong authentication for APIs
- Report vulnerabilities responsibly

❌ **DON'T**:
- Commit API keys or credentials
- Use old/deprecated versions in production
- Skip security updates
- Share vulnerability details publicly
- Exploit vulnerabilities for unauthorized access
