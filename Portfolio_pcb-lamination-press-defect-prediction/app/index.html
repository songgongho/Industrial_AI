MS-CDPNet

Multi-Stage Causal Defect Propagation Network

반도체 PCB MLB Press 공정 이상·불량 예측 및 공정 조건 최적화

Real-Time Defect Detection

Semiconductor PCB lamination (MLB press) processes are complex. Press defects (temperature anomalies, pressure loss, vacuum leaks) occur during the 10+ minute cycle but are only detected after expensive downstream testing.

MS-CDPNet predicts defects in real-time using multimodal sensor data, enabling early intervention and yield optimization.

  • Time-series + event logs + AOI image fusion
  • Causal DAG learning for root cause analysis
  • Cost-aware evaluation (FN weight ≫ FP weight)
  • Built-in explainability (attention maps, SHAP)
  • Production-ready with CI/CD pipeline
🏭

Press Process Monitoring

The Challenge

⚠️ Time Lag

Press defects detected only after expensive downstream testing (24+ hours)

💰 Cost Asymmetry

False negatives (missed defects) → warranty loss (FN_cost=100x)

📊 Data Complexity

Multimodal data (time-series, events, images) with class imbalance (0.03% defect rate)

Our Solution

✅ Real-Time Detection

Predict defects during the press cycle with 95%+ recall, minimizing warranty risk

✅ Explainable AI

Show which variables caused the defect (attention + SHAP) for operator feedback

✅ Cost-Optimized

Threshold tuned for business metrics (FAR@Recall=0.95 < 5%), not just accuracy

Key Features

🧠Multimodal Fusion

Cross-modal attention mechanism for time-series, categorical events, and AOI images

📈Multi-Task Learning

Simultaneous prediction of binary defect, defect type (37 categories), and anomaly confidence

💡Explainability

Attention visualization + SHAP gradients show which features drive predictions

⚙️Cost-Aware Metrics

FAR@Recall=0.95 and cost-weighted evaluation respect business constraints

🤖Synthetic Data Generator

6 P013 anomaly types + multi-anomaly patterns for validation without real data

📊Web Dashboard

Streamlit UI for interactive analysis, data upload, and real-time monitoring

Quick Start (30 seconds)

Get up and running with synthetic demo data:

python -m venv .venv source .venv/bin/activate # or .venv\Scripts\activate on Windows pip install -r requirements.txt python scripts/generate_demo_data.py --output data/demo/sample.parquet python scripts/train.py --data data/demo/sample.parquet --epochs 5 python scripts/predict.py --data data/demo/sample.parquet --checkpoint outputs/model.ckpt python scripts/generate_html_report.py --predictions outputs/predictions.json --output app/dashboard.html

Then open app/dashboard.html in your browser.
