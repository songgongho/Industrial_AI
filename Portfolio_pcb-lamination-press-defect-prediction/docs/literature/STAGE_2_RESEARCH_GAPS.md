# 2단계 | 선행 연구의 공통 한계 → 본 연구의 문제 인식

## 종합 분석: 구조적 공백 (Research Gaps)

선행 연구 5개 흐름을 종합하면, 다음과 같은 **3가지 구조적 공백**이 존재함:

---

## Gap A: 모달 분리 문제 (Modal Fragmentation)

### 현황
```
┌─────────────────────────────────────────┐
│ 시계열 센서 (Press 센서)               │  ← 흐름 1: Susto et al., SECOM
│  ├─ P013: 압력 (4변수)                │
│  ├─ 온도 (PT1~PT9)                    │
│  └─ 진공                               │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│ 이벤트 로그 / 알람                      │  ← 선행 연구: 거의 미활용
│  ├─ Alarm code, severity              │
│  ├─ 설비 가동/비가동                   │
│  └─ 유지보수 이력                      │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│ AOI 이미지 (공정 후 검사)              │  ← 흐름 2: DeepPCB, YOLOv5
│  ├─ 표면 VOID                         │
│  ├─ 휨(Warping)                       │
│  └─ 박리(Delamination)                │
└─────────────────────────────────────────┘
```

### 문제
- **세 정보원은 각각 별도의 독립적 연구로 존재**
- 실제 PCB 적층 Press 공정에서 세 정보는 **동시에 발생하며 상호 보완적**이나, 통합 모델 없음
- 멀티모달 연구(흐름 3)도 **범용 데이터셋** 기반 → 도메인별 물리 특성 미반영

### 본 연구의 접근
✅ **Cross-modal Attention 기반 PressFuse**
- 시계열 + 이벤트 로그를 동시에 처리
- 각 모달의 잠재 표현을 learned attention으로 결합
- Press 공정의 물리적 제약 조건 반영

## Gap B: 사후 탐지 한계 (Post-hoc Detection Limitation)

### 현황: AOI 기반 결함 탐지의 시간축 문제

```
공정 진행 ────────────────────────────────────────→
         
상태      정상  →  정상  →  정상  →  정상  → 불량 ❌
         
검사      발견X    발견X    발견X    발견X  → 발견✓ (너무 늦음)
         
손실      $0      $0      $0      $0     $$$$$$.
                                        (원인 파악 불가)
```

### 문제
- **AOI 이미지 기반 결함 탐지(흐름 2)는 공정 완료 후 발견**
  - VOID, Warping, Delamination는 압착 후 발생 특성
  - Lamination → 베이킹 → 최종 검사까지 수 시간 거리
  
- **공정 중 실시간 예방 불가**
  - 압력/온도/진공 시계열에서 사전 이상 신호를 잡아내는 연구 없음
  - "왜 불량이 발생하는가"를 공정 변수로 설명 불가

### 선행 연구의 한계
- 흐름 1 (센서 PdM): 장비 **고장** 예측에만 국한
  - 제조 **불량**과는 다름
  - SECOM data도 최종 검사 불량만 라벨링
  
- 흐름 4 (인과 DAG): 고장 **원인** 분석은 진행 중이나, 예방 액션 링크 없음

### 본 연구의 접근
✅ **In-process 불량 예측**
- Press 사이클 중간(가압/보온 구간)에서 불량 신호 조기 탐지
- 압력/온도/진공 변수의 비정상 조합 패턴 학습
- 공정 조건 보정 → 불량 모드 원천 차단

✅ **물리 기반 합성 데이터 (synthpress)**
- P013 도메인 지식을 반영한 불량 시나리오 생성
- 실공정 라벨 희소 문제 해결

## Gap C: 라벨 희소 + 비용민감 + 설명 불가 삼중 문제

### 문제 C.1: 극도의 라벨 불균형

**SECOM 데이터 기준:**
- 전체 샘플: 1,567개
- 불량 샘플: 104개 (6.6%)
- 불량 비율: < 7%

```
정상  ████████████████████████████████████████░ (93.4%)
불량  █ (6.6%)
```

**현장(PCB Press) 추정:**
- 통상적 수율: 95~98%
- 불량 비율: 2~5%
- **라벨 희소도: 매우 심각**

**선행 연구의 해결책:**
- SMOTE, under-sampling, class weights (흐름 1, 2)
- **하지만 여전히 증거 부족 → 과적합 위험**

### 문제 C.2: AUROC 단일 지표의 오류

**현장 손실 구조:**
```
FN (미탐, 놓친 불량)
  ├─ 불량 제품 출하 → 고객사 클레임
  ├─ 대규모 리콜 → 수십 배의 손실
  └─ 신뢰도 추락 → 회사 신용 손상

FP (오탐)
  ├─ 정상 제품 폐기 or 재작업
  ├─ 생산 중단 비용
  └─ 손실: 제품 수 배의 비용
```

**비용 비율:** FN_cost : FP_cost = **20 : 1** (또는 그 이상)

**선행 연구의 한계:**
- 흐름 5 (비용민감 학습): 비용 비대칭 **인식**은 있으나
- **실시간 구현**은 극히 드물고
- **멀티모달 맥락**에서는 거의 없음

**단순 AUROC 최적화의 문제:**
```
AUROC=0.95 경우:
- Recall=0.9, Precision=0.8 (20% FP rate)
- Recall=0.8, Precision=0.9 (10% FP rate)

같은 AUROC이지만, 현장 비용은 완전히 다름!
```

### 문제 C.3: 설명 불가 (Black-box)

**현구조:**
```
센서 데이터 (19개 P013 변수)
      ↓
   [모델]
      ↓
   불량/정상 (확률)
      
문제: 왜? → 답 없음
위험: 현장 운영자가 신뢰 불가
```

**선행 연구의 진전:**
- SHAP 기여도 (흐름 5): **어떤 변수가** 기여하는가
- Attention 가중치 (흐름 3): 어느 **시간대가** 중요한가
- DAG (흐름 4): 변수 간 **인과 경로**는?

**하지만:**
- ❌ SHAP만으로는 **시간축 정보 부족**
- ❌ Attention만으로는 **물리적 인과성 약함**
- ❌ **멀티모달 설명** 거의 없음 (왜 이벤트 로그가 중요한지?)

### 본 연구의 접근

✅ **C.1: 합성 데이터 기반 사전학습**
- synthpress 기반 불량 시나리오 생성 (6종 + 4종)
- 라벨 희소 극복
- fine-tune으로 실공정에 적응

✅ **C.2: 이중 비용민감 지표**
- cost_aware_score: FN:FP = 20:1 가중치 손실함수
- FAR@Recall=0.95: 현장 임계값 정책 연동
- 두 지표 **동시 최적화**

✅ **C.3: SHAP + Attention 융합 설명**
- SHAP: 각 변수의 기여도 (어떤 변수)
- Attention: 시간대별 중요도 (언제)
- 이벤트 로그: 물리적 맥락 (왜)
- **세 설명을 cycle 단위로 통합**

## 문제 정의 정리

### 공식 문제 문장
```
PCB 적층 Press 공정에서:

① 멀티모달 정보 (센서/이벤트/이미지)를 
   도메인 특화로 효과적으로 융합하고,

② 공정 **중** 불량을 예방하기 위해 
   사전 신호를 조기에 탐지하되,

③ 라벨 희소(2~5%) + 비용 비대칭(20:1) + 설명 요구(XAI)를 
   동시에 만족하는 방법이 없음.
```

### 연구 필요성 (Motivation)
- **경제성**: 불량 1건 = 수십~수백만 원 손실
- **기술성**: 멀티모달 + 인과 + 비용민감 통합 미진행
- **현장성**: 도메인 특화 모델 필요
- **설명성**: 신뢰도 및 운영 의사결정 지원
