# 공유 물류센터 안전 관리 시스템 개선 프로젝트  
## 데이터 분석 및 Rule-based 라벨링 최종 보고서

### 1. 프로젝트 개요
본 프로젝트는 공유 물류센터 및 공유공장의 실제 센서 데이터(`data_R2.csv`)를 활용하여, 설비의 운영 상태와 안전 이상 징후를 자동으로 분류하는 Rule-based 라벨링 파이프라인을 구축하는 것을 목표로 한다.  
단순히 원시 데이터를 확인하는 수준을 넘어, 설비 운영 중 발생하는 안전 알람, 센서 이상, 비정상 패턴, 유휴 상태를 구분할 수 있도록 상태 라벨을 생성하고, 이를 기반으로 데이터 분석과 시각화를 수행하였다.

### 2. 데이터 및 분석 대상
분석에 사용된 데이터는 총 24,105,658행으로 구성되어 있으며, 주요 컬럼은 다음과 같이 매핑되었다.

- `timestamp` → 2번 열
- `device_id` → 1번 열
- `power` → 16번 열
- `current` → 3번 열
- `voltage` → 6번 열
- `temp` → 23번 열
- `humidity` → 24번 열
- `alarm_flag` → 22번 열

다만 `eqp_status`는 자동으로 인식되지 않았기 때문에, 장비의 명시적 상태 정보에 기반한 유휴 상태 판단은 제한적이었다.

### 3. 라벨링 결과 요약
Rule-based 라벨링을 수행한 결과는 다음과 같았다.

- `alert_state`: 21,427,137건 (88.89%)
- `suspected_sensor_fault`: 2,678,517건 (11.11%)
- `abnormal_pattern`: 2건 (0.00%)
- `idle`: 2건 (0.00%)

이 결과는 전체 데이터에서 `alert_state`가 압도적으로 높은 비중을 차지하고 있음을 보여준다.  
즉, 안전 알람이나 임계치 초과 조건이 매우 자주 발생하고 있거나, `alarm_flag` 기반 규칙이 강하게 작동하고 있다는 의미로 해석할 수 있다.

### 4. 데이터 해석
이번 분석에서 가장 두드러진 특징은 라벨 분포의 극심한 편향이다.  
전체의 88.89%가 `alert_state`로 분류되었고, `suspected_sensor_fault`가 11.11%를 차지하였다. 반면 `abnormal_pattern`과 `idle`은 사실상 거의 발생하지 않았다.

이는 다음과 같이 해석할 수 있다.

1. **안전 알람 또는 이상 신호 빈도 높음**  
   `alarm_flag`가 자주 활성화되었거나, 온도·전력·전류 조건이 경고 임계치를 자주 초과했을 가능성이 높다.

2. **센서 이상 가능성 존재**  
   `suspected_sensor_fault`가 11% 이상으로 나타났으므로, 센서 고착(stuck), 0값 지속, 결측 반복 등의 데이터 품질 문제가 존재할 가능성이 있다.

3. **유휴 상태 탐지 부족**  
   `eqp_status`가 자동 인식되지 않아, 장비 상태 기반의 `idle` 판정이 충분히 반영되지 못했을 수 있다.

4. **비정상 패턴 라벨의 희소성**  
   `abnormal_pattern`이 거의 발생하지 않았기 때문에, 현재 규칙은 강한 알람과 센서 오류는 잘 잡지만, 비알람성 이상 징후에는 다소 보수적으로 동작한 것으로 보인다.

시각화 해석은 다음과 같다.

- **전력-전류 산점도**는 라벨별 군집 차이를 확인할 수 있으며, 센서 이상 및 경고 상태의 분리 가능성을 파악하는 데 유용하다.
- **장비별 타임라인 그래프**는 각 장비의 상태 전환 흐름을 시간 축에서 확인할 수 있어, 특정 장비의 이상 지속 구간을 파악하는 데 도움이 된다.

### 6. 결론
본 프로젝트는 공유 물류센터 안전 관리 데이터를 대상으로 Rule-based 상태 라벨링과 데이터 분석 파이프라인을 성공적으로 구축하였다.  
실제 데이터에 적용한 결과, 대부분의 행이 `alert_state` 또는 `suspected_sensor_fault`로 분류되어, 현장 데이터에 안전 알람 또는 센서 품질 이슈가 상당히 많이 포함되어 있음을 확인하였다.

다만 `eqp_status`가 자동 인식되지 않아 유휴 상태와 정상 가동 상태의 구분이 충분히 반영되지 못했다는 점은 향후 개선이 필요하다.  
향후에는 상태 컬럼 매핑을 보강하고, 규칙 임계치를 재조정하며, supervised learning 기반 분류 모델로 확장함으로써 보다 정교한 이상 탐지 체계를 구축할 수 있다.

### 7. 프로젝트 의의
이 프로젝트는 단순한 데이터 시각화를 넘어,  
1) 실제 운영 데이터에 대한 문제 정의,  
2) 규칙 기반 약 라벨 생성,  
3) 분포 및 패턴 분석,  
4) 향후 머신러닝 확장 가능성 확보  
라는 흐름을 갖춘 포트폴리오형 분석 사례라는 점에서 의미가 있다.
